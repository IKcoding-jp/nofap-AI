## ドキュメントの読み方

### 初めて実装する場合

1. **要件定義書** (`streak_timer_requirements.md`) を読む
   - 機能の目的と要件を理解

2. **実装計画書** (`streak_timer_implementation_plan.md`) を読む
   - 実装の全体計画を把握

3. **技術詳細** (`streak_timer_technical_details.md`) を読む
   - 実装の詳細を理解

4. **チェックリスト** (`streak_timer_checklist.md`) を参照しながら実装
   - タスク管理 (`streak_timer_tasks.md`) で進捗を記録

### 実装中に参照する場合

- **具体的なコード例が必要**: `streak_timer_technical_details.md`
- **次のタスクを確認**: `streak_timer_tasks.md`
- **進捗を記録**: `streak_timer_checklist.md`
- **設計意図を確認**: `streak_timer_implementation_plan.md`

## 実装の流れ

```
1. 要件定義書を読む
   ↓
2. 実装計画書を読む
   ↓
3. 技術詳細を読む
   ↓

   ↓
5. タスク管理で進捗を記録
   ↓

   ↓
7. 完了
```

## 質問・不明点

実装中に不明点が生じた場合は、以下の順序で確認してください:

1. 技術詳細 (`streak_timer_technical_details.md`) の該当セクション
2. 実装計画書 (`streak_timer_implementation_plan.md`) の設計決定
3. 要件定義書 (`streak_timer_requirements.md`) の要件

それでも解決しない場合は、コードベース全体を確認するか、既存の類似実装を参照してください。
