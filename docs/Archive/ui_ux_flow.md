# UI/UX 画面遷移・コンポーネント構成 (UI/UX Flow)

## 1. 画面一覧

### 1.1 ダッシュボード (`/`)
- **主な役割**: 現在の状態の確認とクイックアクション。
- **コンポーネント**:
  - `StreakCounter`: 現在の継続日数を巨大な数字で表示。
  - `StatusButtons`: 「今日の成功/失敗」を記録するボタン。
  - `MoteMeter`: モテ度をゲージで表示。
  - `MotivationalQuote`: AI生成または固定の励ましメッセージ。

### 1.2 カレンダー (`/calendar`)
- **主な役割**: 過去の記録の視覚化。
- **コンポーネント**:
  - `ActivityCalendar`: 青丸（成功）と赤丸（失敗）が表示されたカレンダー。
  - `DayDetailsSheet`: 日付クリック時に詳細を表示するサイドパネル。

### 1.3 日記 (`/journal`)
- **主な役割**: 反省と振り返りの一覧。
- **コンポーネント**:
  - `JournalList`: タイムライン形式の日記一覧。
  - `JournalEditor`: 簡易的なテキストエディタ。

### 1.4 AIチャット (`/chat`)
- **主な役割**: AIへの相談とメンタルケア。
- **コンポーネント**:
  - `ChatWindow`: メッセージの履歴表示。
  - `ChatInput`: AIへのメッセージ送信フォーム。
  - `QuickActions`: 「誘惑に負けそう」「アドバイスが欲しい」等のクイックボタン。

## 2. ユーザー体験 (UX) のこだわり
- **ポジティブなフィードバック**:
  - ストリークが更新された際のアニメーション。
  - モテ度メーターが上昇した際の演出。
- **失敗時のケア**:
  - 失敗を記録した際、AIが優しく寄り添い、再起を促すメッセージを表示。
  - 「なぜ失敗したか」を言語化させるステップを設け、改善に繋げる。

## 3. スタイルガイド
- **カラーパレット**:
  - 成功: `blue-500` (清潔感、冷静さ)
  - 失敗: `red-500` (注意喚起、情熱の再燃)
  - ベース: `slate-900` または `white` (シンプルで集中できる環境)
- **フォント**: Sans-serif (可読性重視)
